My Complete Git Learning Journey

A Detailed Documentation of My Step-by-Step Git and GitHub Learning Process

Overview

This page documents my complete journey of learning Git and GitHub — starting from basic terminal usage to building a professional web-based documentation of version control. It includes every step, command, and problem I encountered, along with how I resolved each one.

1. Installation and Setup

Checking if Git is Installed

git --version

Confirmed that Git was available on my Mac. If not, it could be installed via Xcode or Homebrew.

Learning Basic Terminal Commands

cd <folder_name>
ls -a

Learned to navigate folders and understand the concept of the working directory and hidden .git folder.

Initializing a Repository

git init

Created a new local repository. Learned that this command should only be used once per project to avoid reinitialization.

Configuring Git User Details

git config --global user.name "Rakesh8050"
git config --global user.email "125812375+Rakesh8050@users.noreply.github.com"

Set up author information that appears in each commit.

2. Core Git Commands and Workflow

Adding and Committing Files

git add .
git commit -m "Added c1, c2, and c3"

Learned how staging and committing work. Understood that “add” moves files to staging and “commit” saves them permanently.

Checking Repository Status

git status

Interpreted untracked, modified, and committed files.

Viewing Commit History

git log

Understood commit IDs, timestamps, and author details.

Reverting and Resetting Commits

git revert <commit-id>

Used to safely undo a previous commit by creating a new one.

Exploring Differences

git diff
git diff c3.txt

Compared changes between commits and file versions using plus/minus indicators.

Branching and Switching

git branch
git checkout <branch>

Complete Workflow Summary


# Initialize repository
git init

# Stage files
git add .

# Commit changes
git commit -m "message"

# Push to GitHub
git push -u origin main

3. Troubleshooting and Common Issues

1. Merge Conflicts

CONFLICT (modify/delete): c3.txt deleted in parent and modified in HEAD

Resolved manually by editing files, adding with git add, and continuing the revert process with git revert --continue.

2. “Invalid Username or Token” Error

Learned that GitHub no longer supports password-based pushes. Created a Personal Access Token (PAT) with:

  • All repositories access
  • Contents: Read & Write
  • Metadata: Read-only

3. Push Rejection (Fetch First)

! [rejected] main -> main (fetch first)

Resolved with:

git fetch origin
git pull origin main --rebase
git push origin main

4. Branch Name Confusion (main vs master)

error: src refspec master does not match any

Fixed by identifying the active branch and pushing correctly:

git branch
git push -u origin main

5. “Everything Up to Date” Issue

Encountered when new files weren’t staged or committed.

git add .
git commit -m "Added new file"
git push

4. Documentation and Professional Writing

Creating a README.md

Wrote a complete Markdown file covering:

  • Overview
  • Setup and Prerequisites
  • Command explanations
  • Example workflows
  • Common errors and notes

Repository Setup

git remote remove origin
git remote add origin https://github.com/Rakesh8050/git-for-beginners.git
git push -u origin main

Linked a new GitHub repository titled git-for-beginners.

5. Building the Git Learning Web Page

Converted README.md content into an HTML webpage (index.html) featuring:

  • Blue and white professional theme
  • Navigation bar for sections
  • Footer with credits
  • Code highlighting with preformatted blocks

Added Sections:

  • Overview
  • Setup
  • Uploading Files
  • Commands Reference
  • Troubleshooting
  • Learning Challenges and Solutions

Final Push

git add index.html
git commit -m "Added complete learning journey webpage"
git push origin main

Successfully uploaded the final web version to GitHub.

6. Final Outcome and Understanding

  • Mastered the complete Git workflow: init → add → commit → push → pull → revert
  • Understood branching, merging, and remote synchronization
  • Resolved real-world Git errors and merge conflicts
  • Created a professional README and HTML-based documentation
  • Built a public learning portfolio: git-for-beginners

This project demonstrates the end-to-end process of learning, applying, and documenting Git for version control in a professional and structured manner.
