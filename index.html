    <h2 id="overview">Overview</h2>

    <p>

      Git is a distributed version control system used to track changes in files and collaborate efficiently. 

      It allows users to maintain multiple versions of their work, restore previous states, and manage contributions easily.

    </p>

    <p>

      This guide explains how to set up Git, connect to GitHub, and push files to a remote repository using the command line.

    </p>


    <hr>


    <h2 id="setup">Initial Setup</h2>


    <h3>1. Initialize Git in your local project folder</h3>

    <pre><code>git init</code></pre>


    <h3>2. Configure Git user details</h3>

    <pre><code>git config --global user.name "Your Name"

git config --global user.email "youremail@example.com"</code></pre>


    <h3>3. Connect your folder to GitHub</h3>

    <pre><code>git remote add origin https://github.com/&lt;your-username&gt;/&lt;repository-name&gt;.git</code></pre>


    <h3>4. Generate a Personal Access Token</h3>

    <p>Since GitHub no longer accepts passwords for Git operations, you must use a Personal Access Token (PAT).</p>

    <ol>

      <li>Go to <a href="https://github.com/settings/tokens" target="_blank">GitHub Tokens</a></li>

      <li>Select <b>Fine-grained personal access token</b></li>

      <li>Set repository access to <b>All repositories</b></li>

      <li>Enable:

        <ul>

          <li>Contents: Read and Write</li>

          <li>Metadata: Read-only</li>

        </ul>

      </li>

    </ol>


    <hr>


    <h2 id="upload">Steps to Upload Files to GitHub</h2>


    <h3>1. Check current status</h3>

    <pre><code>git status</code></pre>


    <h3>2. Add files to the staging area</h3>

    <pre><code>git add filename.txt

git add .</code></pre>


    <h3>3. Commit changes</h3>

    <pre><code>git commit -m "Added new files and updates"</code></pre>


    <h3>4. Push to GitHub</h3>

    <pre><code>git push -u origin main</code></pre>


    <hr>


    <h2 id="commands">Commonly Used Git Commands</h2>

    <table>

      <tr><th>Command</th><th>Description</th></tr>

      <tr><td>git status</td><td>Check current state of files</td></tr>

      <tr><td>git add .</td><td>Stage all new and modified files</td></tr>

      <tr><td>git commit -m "message"</td><td>Save staged changes locally</td></tr>

      <tr><td>git push</td><td>Upload commits to GitHub</td></tr>

      <tr><td>git pull</td><td>Fetch and merge updates from GitHub</td></tr>

      <tr><td>git log</td><td>Display commit history</td></tr>

      <tr><td>git revert &lt;commit-id&gt;</td><td>Undo a specific commit</td></tr>

      <tr><td>git branch</td><td>List all branches</td></tr>

      <tr><td>git checkout &lt;branch&gt;</td><td>Switch between branches</td></tr>

    </table>


    <hr>


    <h2 id="workflow">Example Workflow</h2>

    <pre><code># View current changes

git status


# Stage all modified files

git add .


# Commit with a message

git commit -m "Updated project files"


# Push to GitHub

git push</code></pre>


    <h2>Notes</h2>

    <ul>

      <li>“Everything up to date” means no new changes to push.</li>

      <li>Always follow this sequence: <b>add → commit → push</b>.</li>

      <li>One token can be used for all repositories if created with “All repositories” access.</li>

      <li>Run <code>git pull</code> before pushing to stay synced with GitHub.</li>

    </ul>
