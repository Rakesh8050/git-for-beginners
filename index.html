Overview

This page documents my complete learning journey with Git — from basic installation to creating and publishing a professional Git guide as a web page. It includes every command, concept, and problem encountered during the process, along with how each was resolved.

Learning Steps and Key Milestones

1. Checking if Git Was Installed

git --version

Verified installation and understood Git as a version control tool.

2. Learning Basic Terminal Commands

cd <folder_name>
ls -a

Learned how to navigate directories and understood the working directory concept.

3. Creating and Initializing Repositories

git init

Created a new repository and learned about the .git folder.

4. Adding and Committing Files

git add .
git commit -m "Initial commit"

Understood staging and committing changes in Git.

5. Checking Repository Status

git status

Learned to interpret “untracked files,” “changes to be committed,” and “working tree clean.”

6. Viewing Commit History

git log

7. Reverting and Resolving Conflicts

git revert <commit-id>
git add .
git revert --continue

Faced merge conflicts and learned how to manually fix them.

8. Connecting to GitHub

git remote add origin https://github.com/Rakesh8050/demo.git
git push -u origin master

9. Authentication with Personal Access Token

Learned that GitHub no longer supports passwords and created a Personal Access Token (PAT) for secure access.

10. Switching to a New Repository

git remote remove origin
git remote add origin https://github.com/Rakesh8050/git-for-beginners.git

11. Handling Branch Confusion (main vs master)

git branch
git push -u origin main

12. Fixing Push Rejections

git fetch origin
git pull origin main --rebase
git push origin main

13. Writing a Professional README.md

Structured and formatted documentation covering setup, configuration, workflow, and troubleshooting.

14. Building a Web Page (index.html)

Converted the README into a professional webpage with a blue-white theme and navigation bar.

Complete Git Workflow Learned


# Initialize repository
git init

# Stage files
git add .

# Commit changes
git commit -m "message"

# Push to GitHub
git push -u origin main

Also learned advanced commands like git pull, git revert, and git reset.

Troubleshooting Common Git Errors

Push Rejection (fetch first)

! [rejected] main -> main (fetch first)

Resolved by syncing local and remote branches:

git fetch origin
git pull origin main --rebase
git push origin main

Branch Mismatch (master vs main)

error: src refspec master does not match any

Resolved by pushing to the correct branch:

git push -u origin main

Invalid Password or Token Error

Invalid username or token. Password authentication is not supported.

Created a Personal Access Token (PAT) and used it instead of a password.

  • Understood all key commands and branching strategies
  • Built and deployed a professional guide webpage
  • Documented all learning challenges and solutions
