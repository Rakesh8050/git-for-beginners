Overview

Git is a distributed version control system used to track changes in files and collaborate effectively. It allows you to maintain multiple versions, restore previous states, and manage updates between local and remote repositories.

This guide provides a complete, step-by-step workflow for initializing repositories, configuring Git, adding files, committing changes, and pushing updates to GitHub.

Initial Setup

1. Initialize Git in your project folder

git init

2. Configure your user credentials


git config --global user.name "Your Name"
git config --global user.email "youremail@example.com"

3. Connect to your remote repository

git remote add origin https://github.com/<username>/<repository>.git

4. Create a Personal Access Token (PAT)

Since GitHub no longer allows password-based authentication, you must use a Personal Access Token for command-line operations.

  1. Go to GitHub Tokens
  2. Generate a new fine-grained token with repository access
  3. Enable Contents (Read and Write) and Metadata (Read-only)
  4. Copy the token securely and use it as your password when pushing

Uploading Files to GitHub

Step 1: Check status

git status

Step 2: Add files to staging

git add .

Step 3: Commit changes

git commit -m "Added project files"

Step 4: Push to GitHub

git push -u origin main

Example Workflow


git status
git add .
git commit -m "Updated files"
git push

Always follow the sequence: add → commit → push for smooth synchronization between your local and remote repositories.

Troubleshooting Common Git Errors

Error Message:

! [rejected] main -> main (fetch first)
error: failed to push some refs to 'https://github.com/<user>/<repo>.git'

Cause: The remote branch has new commits not present locally.

Fix:


# Fetch remote updates
git fetch origin

# Rebase local commits on top of remote
git pull origin main --rebase

# Push updated branch
git push origin main

If the issue persists and you’re certain your local version is correct: 

git push origin main --force

Note: Use --force cautiously; it overwrites the remote branch.
